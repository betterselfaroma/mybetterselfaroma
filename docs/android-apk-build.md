# Android APK Build Guide

This guide prepares the **香气读懂你的心 Admin** PWA for Android packaging with Bubblewrap / Trusted Web Activity.

## App Settings

- App name: 香气读懂你的心 Admin
- Short name / launcher name: Scent Admin
- Package name: `com.betterselfaroma.admin`
- Start URL: `https://scentknowsyou.com/admin`
- Scope: `/admin`
- Theme color: `#2f5d46`
- Background color: `#f7f1e6`
- Orientation: `portrait`

## A. Prerequisites

- The website must already be deployed on HTTPS.
- `https://scentknowsyou.com/manifest.webmanifest` must be accessible.
- `https://scentknowsyou.com/service-worker.js` must be accessible.
- `https://scentknowsyou.com/icons/icon-192.png` must be accessible.
- `https://scentknowsyou.com/icons/icon-512.png` must be accessible.
- Android Studio must be installed.
- Node.js must be installed.
- Java / JDK must be installed.
- Admin login, staff/admin permissions, camera scan, and PWA install behavior should be tested on the live HTTPS domain first.

## B. Recommended Packaging Method

Use **Bubblewrap** to generate a Trusted Web Activity (TWA). This wraps the existing HTTPS Admin PWA in an Android app shell while keeping the app content served by `https://scentknowsyou.com/admin`.

## C. Install Bubblewrap

```bash
npm install -g @bubblewrap/cli
```

Then confirm it is available:

```bash
bubblewrap --version
```

## D. Initialize TWA

Run:

```bash
bubblewrap init --manifest https://scentknowsyou.com/manifest.webmanifest
```

When prompted, use:

- Package name: `com.betterselfaroma.admin`
- App name: `香气读懂你的心 Admin`
- Launcher name: `Scent Admin`
- Start URL: `https://scentknowsyou.com/admin`
- Theme color: `#2f5d46`
- Background color: `#f7f1e6`
- Display mode: `standalone`
- Orientation: `portrait`

## E. Digital Asset Links

TWA release builds need Digital Asset Links verification.

The template file lives at:

```text
public/.well-known/assetlinks.json
```

Before a real release, replace:

```text
TODO_REPLACE_WITH_RELEASE_KEY_SHA256
```

with the SHA256 fingerprint of the Android signing certificate generated by Bubblewrap / Android Studio.

Do not publish a final Play Store release with the TODO fingerprint still in place.

## F. Build Debug APK

After `bubblewrap init`, run:

```bash
bubblewrap build
```

Bubblewrap will generate a debug APK inside the generated Android project output, usually under an app build output folder such as:

```text
app/build/outputs/apk/debug/
```

Exact paths may differ depending on the Bubblewrap version and generated project layout.

## G. Android Phone Install Test

1. Copy the debug APK to an Android phone.
2. Enable install from unknown sources for the file manager or browser used.
3. Install the APK.
4. Open `Scent Admin`.
5. Confirm it starts at `https://scentknowsyou.com/admin`.
6. Log in as an admin or staff account.
7. Confirm dashboard, bookings, members, points, and settings load correctly.
8. Open `/admin/scan`.
9. Tap the scan button and approve camera permission.
10. Scan a member QR code.

Camera scanning requires HTTPS in production. It may also work on `localhost` for development, but the APK should be tested against the live HTTPS domain.

## H. Build Release / AAB

For a Google Play release, build a release bundle:

```bash
bubblewrap build --mode=release
```

Bubblewrap / Android tooling will ask for or generate signing information. After the release key is ready, update:

```text
public/.well-known/assetlinks.json
```

with the real SHA256 certificate fingerprint, deploy the site, then verify that:

```text
https://scentknowsyou.com/.well-known/assetlinks.json
```

is accessible before Play Store submission.

## I. Final Release Checklist

- `npm run build` passes.
- Admin PWA is deployed to HTTPS.
- Manifest and service worker are accessible.
- `assetlinks.json` contains the real release SHA256 fingerprint.
- Admin/staff login works.
- Regular members cannot access Admin routes.
- Camera scan works on a real Android device.
- Points adjustments write `points_transactions`.
- Admin actions write `admin_audit_logs`.
- The app opens directly to `/admin` from the launcher.
